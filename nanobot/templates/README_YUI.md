# Yui 核心引擎：运行区 (Workspace) - 深度解析

本模块是 Yui 运行时的“实体店”，存放着它的性格、记忆、工具说明及主动任务列表。

## 核心组件架构

### 1. 灵魂与准则 (Soul & Agents)
- **`SOUL.md`**: 定义 Yui 的人格特征（如：友好、简洁、好奇）。
- **`AGENTS.md`**: 定义操作规范。包含：
    - **解释权**: 规定 Agent 在行动前必须先解释。
    - **工具调用逻辑**: 如何添加提醒事项、如何处理 `HEARTBEAT.md`。

### 2. 工具说明书 (`TOOLS.md`)
- **机器可读**: 这是一份专门喂给 LLM 的 Markdown 格式文档。
- **SOP 教学**: 提供了大量的 `nanobot cron add` 命令行示例，教导 Agent 如何在没有 Python API 的情况下，通过 `exec` 自主调度任务。

### 3. 记忆系统 (`memory/`)
- **`MEMORY.md`**: **长期记忆 (Key Facts)**。采用结构化模板（用户信息、偏好、项目上下文）。由 Agent 定期整理并持久化。
- **`HISTORY.md`**: **短期记忆日志 (Log)**。记录每一轮对话和工具执行的流水账，供 `grep` 检索。

### 4. 主动任务区 (`HEARTBEAT.md`)
- **工作模式**: 这是一个 Markdown 列表。Agent 每 30 分钟会醒来读一次。
- **自主维护**: Agent 会根据任务进展，自主执行 `edit_file` 来勾选或删除任务。

## 开发者备忘 (Yui 魔改点)
- **人格定制**: 这是我们实现“野蛮生长”的第一站。可以直接重写 `SOUL.md` 和 `AGENTS.md` 来注入 Yui 的核心目标。
- **技能预埋**: 任何新开发的 Skill，都应该在 `TOOLS.md` 中补充对应的操作示例，以便 Agent 快速上手。
- **记忆迁移**: 你历史存量的知识库内容，可以直接批量导入到 `MEMORY.md` 的对应章节下。
